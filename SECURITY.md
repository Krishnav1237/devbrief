# Security Policy

We take the security of DevBrief seriously. If you believe you have found a security vulnerability or threat in DevBrief (or in any of its dependencies/scanners), please do not report it in a public issue.

Instead, please send an email to the maintainers or report it privately.

## Supported Versions

Only the latest major/minor release is actively supported with security updates.

| Version | Supported |
| --- | --- |
| >= 0.1.x | Yes |
| < 0.1.0 | No |

## Reporting a Vulnerability

1. Report the vulnerability via the GitHub Repository Security tab (Private Vulnerability Reporting) or by emailing security@devbrief.io.
2. Include reproduction steps, sample payloads, or code snippets.
3. We will acknowledge receipt of the report within 48 hours and work with you to remediate it.
